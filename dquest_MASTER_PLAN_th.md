## 1. วิสัยทัศน์โดยย่อ
D-Quest = เว็บแอป gamification ส่งเสริมการทำความดีของนักศึกษา NICEC (472 คน)

**วงจรหลัก:** ทำดี → ถ่ายรูป+อธิบาย → ส่ง → **อาจารย์/admin ตรวจ (มนุษย์เสมอ)** → อนุมัติ → ได้ D Score + D Coin → กระดานผู้นำ / เอา D Coin ไปสุ่มกาชา → ปลดล็อกของแต่งตัว

**กลยุทธ์ทับทุกอย่าง (จากคุณ):** ทำ **base ให้แน่นก่อน หยั่งเชิงความนิยมก่อน** ถ้าฮิตค่อยใส่สุด → **ทุกฟีเจอร์หรูเป็น "วางฐานเผื่ออนาคต" ไม่ใช่ทำเต็มตอนนี้**

**Non-negotiables:** ให้คะแนนผ่านมนุษย์เสมอ · บัญชี=ชื่อจริง ตรวจย้อนได้ · ฟรีหรือถูกมาก · ทางการสไตล์สถาบัน + สนุกแบบเกม

**3 แนวคิด "avatar" ที่ต้องแยกกัน:**
1. **ตัวละครยืนหน้า home** = สไตล์ Gacha Life (2D) เลือกชุดตามแผนก + ล็อกเพศตอนสมัคร — ตอนนี้ทำ **base ขั้นต่ำ**
2. **รูปโปรไฟล์บัญชี** = **avatar เทาเหมือนกันหมด ยังไม่มีระบบเปลี่ยน**
3. **Snapshot** = ภาพตัวละคร + displayname "แช่แข็ง" ตอนจบซีซั่น → Hall of Fame (ดู §5)

**ต่างจาก Brawl Stars:** ยืมแค่เลย์เอาต์ — **ไม่มีปุ่ม PLAY/จับคู่แมตช์** เพราะ home คือฮับ

## 3. ข้อมูลจริงจาก TSV

### 3.1 students.tsv — 472 คน
`studentCode` · `prefix` · `name` · `fullname` · `program` · `department` · `level` · `groupCode` · `groupName` · `advisor` · `ptype`
- **คำนำหน้า → ล็อกเพศสะอาด 100%:** ชาย 285 · หญิง 187 → ไม่มีเคสกำกวม
- **`studentCode`** = PK + ใช้แก้ชื่อซ้ำตอนค้นหา
- **`advisor`** เก็บแบบมีคำนำหน้า → **ตรงกับ `staff.fullname` เป๊ะ (match key: student.advisor = teacher.fullname)**
- **48 กลุ่มเรียน** · ปวช 349 / ปวส 123

### 3.2 staff.tsv — 45 คน
ผอ. = นายสุนทร เรืองวัฒนานนท์ · รองผอ. = นางสาวคณัสนันท์ ศรีเกียรติ, นางแอนนา ผลไสว
- **map role:** ผู้บริหาร+ครู → teacher · admin = ผอ./รองผอ. (3) · เจ้าหน้าที่ = staff

### 3.3 login แบบ B + `advisor`
1. นักศึกษาค้นชื่อ → ถ้าซ้ำ โชว์ groupName ให้เลือกถูกคน
2. กดอ้างสิทธิ์ → "รออนุมัติ"
3. ครูที่ปรึกษา (หรือครู/admin คนใด) กดอนุมัติ → เปิดใช้งาน
4. คิวอนุมัติ **filter ตาม advisor** → ครูเห็นเฉพาะลูกศิษย์ + ปุ่ม **"อนุมัติทั้งกลุ่ม"** — **(ทำแล้ว 20 มิ.ย. · มีปุ่ม "ดูทั้งหมด" ช่วยกลุ่มอื่นได้)**
- บุคลากร/admin = login จริง (email+password) แยกจาก flow นักศึกษาเสมอ

## 7. เศรษฐกิจ
- **D Score** (แข่ง/รีเซ็ต) + **D Coin** (สะสม/ถาวร) · 1 ความดีอนุมัติ = +1 ทั้งคู่ 🔒
- ค่าคะแนน: ขยะเล็ก 1–3 · ส่วนรวม 3–8 · ของมีค่า/กุญแจ 10–20 · พิเศษ 20–50 (ดุลยพินิจครู)
- กันโกง: daily_cap ใน deed_types · คูลดาวน์ · ครูมองภาพซ้ำเอง
- **เช็คอินรายวัน:** ให้ **D Coin อย่างเดียว ไม่แตะ D Score** · streak 7 วัน (10,10,15,15,20,20,50)
- **กาชา:** ราคา **10 D Coin** · 4 ระดับถ่วงน้ำหนัก · ได้ซ้ำ **คืนเหรียญ** (2/4/6/8) 🔒 (ปรับ 20 มิ.ย.)
