# D-Quest — รายงานวิจัย "ออกนอกกะลา" รอบที่ 1

**วันที่:** 20 มิถุนายน 2026 · **โดย:** ผู้ช่วย AI (ตามที่ภูมิมอบหมายให้ไปค้นคว้าเชิงลึก)
**ขอบเขต:** ระบบ Gacha Life 2 (แต่งตัว/โพส/สตูดิโอ/กาชา) + กลไกกาชาในเกมจริง + เทคนิคทำ avatar บนเว็บ + งานวิจัย gamification/แรงจูงใจ + บทเรียนจากแอปคู่แข่งในโลกจริง + ความปลอดภัยรูปภาพ

> **อ่านอย่างน้อย §0 และ §6** — สองส่วนนี้กระทบ "หัวใจ" ของ D-Quest มากที่สุด

## 0. บทสรุปผู้บริหาร — 5 สิ่งที่ต้องรู้

1. **⚠️ รางวัลวัตถุอาจ "ทำลาย" การทำดี (Overjustification Effect)** — งานวิจัยพบว่าเด็กที่ได้ "รางวัลสิ่งของ" สำหรับการช่วยเหลือ กลับ **ช่วยน้อยลง** ในภายหลัง เทียบกับเด็กที่ได้แค่ "คำชม" หรือไม่ได้อะไรเลย เพราะสมองเปลี่ยนจาก "ฉันทำเพราะอยากทำ" → "ฉันทำเพื่อรางวัล" · **นี่คือความเสี่ยงตรงหัวใจ D-Quest** (แอปที่จ่าย D Coin/D Score ให้การทำดี) → ต้องออกแบบให้รางวัลเป็น **"การยอมรับ/ให้ข้อมูล" ไม่ใช่ "การจ้าง"** (ดู §6)

2. **gamification ได้ผลจริง ถ้าตอบ 3 ความต้องการพื้นฐาน** (Self-Determination Theory): **Autonomy** (เลือกเอง), **Competence** (เก่งขึ้น/เชี่ยวชาญ), **Relatedness** (เป็นส่วนหนึ่งของกลุ่ม) · meta-analysis ยืนยันว่า gamification เพิ่มแรงจูงใจภายใน + ความรู้สึก autonomy/relatedness ได้จริง

3. **ClassDojo (แอปให้แต้มพฤติกรรมที่ดังที่สุดในโลก) โดนวิจารณ์หนัก** ใน 4 เรื่อง: ประจานต่อหน้าคนอื่น, สอดส่อง (เด็กกล้าเสี่ยงน้อยลง), สร้าง "พฤติกรรมแสร้งทำเพื่อแต้ม", และอคติทางวัฒนธรรม · D-Quest หลบได้หลายข้อ (ให้แต้มเฉพาะด้านบวก, ไม่หักต่อหน้า) แต่ต้องระวังเรื่อง "ประจาน" บนกระดานและ "แสร้งทำ"

4. **กาชา Fragment/Pity ในสเปกเราสอดคล้องมาตรฐานอุตสาหกรรม** — Genshin ใช้ soft pity ~74, hard pity 90, base 0.6% · ของเรา (UR 0.9%, hard 200+) สมเหตุสมผล · **กุญแจคือต้องโชว์ตัวนับ pity ให้เห็น** ("อีก X ครั้งได้แน่")

5. **ภาชนะตัวละครเรามาถูกทางแล้ว** — GL2 ใช้แนวคิดเดียวกับที่เราวาง: layer + แยกผมหน้า/หลัง/ข้าง + ทุกชิ้นปรับ position/scale/rotation/flip + color slider/palette · มีไลบรารีเปิด (DiceBear) ที่ทำ avatar แบบเลเยอร์ SVG บนเว็บ ใช้เป็นต้นแบบเทคนิคได้

## 1. Gacha Life 2 — ระบบแต่งตัวละคร (เจาะลึก)

GL2 = เกมที่ "ทั้งเกมคือการแต่งตัว" สิ่งที่ทำได้จริง:

**โครงสร้างชิ้นส่วน (ละเอียดกว่าที่เราคิด):**
- **ผมแยกหลายชั้น:** ผมหน้า (front), ผมหลัง (rear/back), ผมข้างซ้าย, ผมข้างขวา (แยกซ้าย-ขวาได้อิสระ!) — ผสมกันได้หลายชิ้นพร้อมกัน
- **ตา:** แยก eye shape + eyeball + pupil + เส้น eyelash ปรับแยกกัน
- **เสื้อผ้า:** ปรับ **ความยาวแขนเสื้อ/ถุงเท้า** ได้ · ปรับรองเท้าและกางเกง **ทีละข้าง/ทีละส่วน**
- **อุปกรณ์เสริมมี layer เฉพาะ:** หมวก, แว่น, ผ้าคลุม (cape), หาง (tail), ปีก (wings)
- **Special FX:** เพิ่มเอฟเฟกต์ เช่น แสงเรือง (glow)

**การปรับแต่งต่อชิ้น (Adjustments) — ทุกชิ้นทำได้:**
- เลื่อนอิสระ (free move) หรือใช้ลูกศร
- **X scale, Y scale, Rotation**
- **Flip แนวนอน/แนวตั้ง**
- ปุ่ม **chevron `< ● >`** สลับ asset / สลับซ้าย-ขวา
- **Pose Layers:** ดันชิ้นส่วน "ไปข้างหน้า/ข้างหลัง" (ปรับ z-order ต่อชิ้น)

**สี:**
- **Color Slider** (สีจำกัด ปรับเร็ว) ↔ สลับไป **Color Palette** (เฉดเยอะ) — มีปุ่มสลับมุมซ้ายบน
- ปรับสีได้ทุกส่วน/ทุกชิ้น

**→ สรุปสำหรับเรา:** สถาปัตยกรรม slot/layer + transform ต่อชิ้น ที่เราเพิ่งวางไว้ **คือทิศทางเดียวกับ GL2 เป๊ะ** สิ่งที่ต้องเติมตามลำดับ: (1) แยกผมข้างซ้าย/ขวา + ตา-ลูกตา-รูม่านตาแยกชั้น, (2) UI ปรับ X/Y/scale/rotate/flip ต่อชิ้น, (3) toggle slider↔palette, (4) ความยาวแขน/ถุงเท้าแบบปรับได้

## 2. Gacha Life 2 — Pose / Animation / Studio

- **Pose Editor:** ปรับข้อต่อร่างกายทีละส่วน — upper/lower left/right hand, upper/lower left/right leg, นิ้วซ้าย/ขวา, torso, head · เลื่อน/หมุน/ย่อขยายแต่ละส่วน
- **โพสสำเร็จรูปหลายร้อยท่า** + **สร้างโพสเองได้**
- **Export/Import เป็น "Pose Code" (ข้อความ)** — ก๊อปโค้ดไปวางใส่ตัวอื่น/แชร์ได้ ← เทคนิคนี้ฉลาดมาก (โพส = สตริงสั้นๆ พกพาง่าย)
- **Studio Mode:** วางได้สูงสุด **8 ตัวละคร** ในฉากเดียว · เลื่อน/ย่อ/หมุน/flip · **Lock-On** ตรึงตำแหน่งกันเผลอขยับ · **600+ พื้นหลัง** · เพลงล็อบบี้ 6 เพลง
- **ข้อจำกัดปัจจุบันของ GL2:** "animated pose" ยังใช้ใน Studio โดยตรงไม่ได้ (เขายังพัฒนาอยู่)

**→ สรุปสำหรับเรา:** Pose/Studio เป็น **เฟสปลายทางมาก** (เกินความจำเป็นของแอปทำความดี) แต่แนวคิด **"โพส = โค้ดข้อความ export/import ได้"** เอามาใช้กับ **ทั้งตัวละคร** ของเราได้เลย — `avatar_key` ของเราก็คือ "โค้ดตัวละคร" อยู่แล้ว ทำปุ่ม "คัดลอก/แชร์ลุค" ได้ในอนาคต

## 3. กลไกกาชาในเกมจริง (เทียบสเปก Custom.txt ของเรา)

**Gacha Life 2 เอง:** 10 เจม/พุล, 100 เจม/มัลติ(10) · 1–6 ดาว · 550 ยูนิต · เจมจาก mail/โฆษณา · ไม่มีระบบต่อสู้ (สะสมอย่างเดียว)

**Genshin Impact (มาตรฐาน pity ที่คนอ้างอิงมากสุด):**
- base rate 5★ = **0.6%**
- **Soft pity** เริ่ม ~พุล 74–76 → โอกาสพุ่งขึ้นทุกพุล
- **Hard pity** = 90 (รับประกัน)
- **50/50:** ถ้าได้ 5★ แต่ "แพ้ 50/50" จะได้ตัว standard → ครั้งหน้ารับประกันตัว featured
- **pity ข้ามแบนเนอร์** (limited→limited) แต่ไม่ข้ามไป standard

**เทียบกับ Custom.txt ของเรา (UR 0.9%, SSR 2.5%, soft เริ่ม 45, hard SSR ~85-95 / UR ~200-220, Refinery หลอมเศษ):**
- ✅ ตัวเลขเรา**อยู่ในเกณฑ์มาตรฐาน** (ใกล้ Genshin, ใจป้ำกว่านิดด้วยซ้ำที่ soft เริ่มเร็ว=45)
- ✅ **Refinery (หลอมเศษระดับต่ำ→สูง)** เป็นจุดเด่นที่ Genshin ไม่มี → แก้ปัญหา "เศษกาก" ได้ดี เป็นไอเดียที่ดีจริง
- 📌 **ต้องมี: ตัวนับ pity ที่มองเห็น** ("อีก 37 พุลได้ SSR แน่") — ทุกแหล่งย้ำว่านี่คือสิ่งที่ทำให้ผู้เล่นไม่สิ้นหวัง/ไม่รู้สึกโดนหลอก
- 📌 **ข้อคิด:** ระบบเรา **ไม่มีเงินจริง** (ได้ D Coin จากทำดี/เช็คอิน) → ความกดดันแบบกาชาเงินจริงไม่มี ดังนั้นเราออกแบบให้ "ใจดี" กว่าเกมพาณิชย์ได้ และ **ไม่ควรลอกความโหดของ rate กาชาจ่ายเงินมาตรงๆ**

## 4. เทคนิคทำ Avatar 2D บนเว็บ

**DiceBear** (โอเพนซอร์ส, เป็นมาตรฐานของวงการ):
- เปลี่ยน seed → SVG avatar, 35+ สไตล์

- ถ้าจะไป "อนิเมะกว่า" แบบ GL2 จริงๆ → ต้องมี **asset ภาพ (PNG/SVG วาดมือ)** หลายชิ้นมาประกอบ ซึ่งคือ "งานวาด" (จ้างศิลปิน/หา asset ลิขสิทธิ์เปิด) ไม่ใช่แค่โค้ด — **นี่คือข้อจำกัดจริง** ที่ต้องตัดสินใจ: วาดเอง/จ้าง/ใช้ asset เปิด vs คงสไตล์ geometric ที่โค้ดสร้างได้เอง

## 5. Gamification เพื่อการศึกษา (หลักฐานวิจัย)

**Self-Determination Theory (SDT)** = กรอบที่งานวิจัยใช้มากสุด · แรงจูงใจภายในงอกงามเมื่อได้ 3 อย่าง:
- **Competence** — รู้สึกเก่งขึ้น เอาชนะความท้าทาย
- **Autonomy** — รู้สึกได้เลือกเอง มีอิสระ ไม่ถูกบังคับ
- **Relatedness** — รู้สึกเชื่อมโยงกับคนอื่น/กลุ่ม

**หลักฐาน:**
- meta-analysis: gamification **เพิ่มแรงจูงใจภายใน + ความรู้สึก autonomy/relatedness** ได้จริง แต่ **ผลต่อ "ความสามารถจริง (competency)" น้อย**
- gamification แบบ challenge-based เพิ่มผลสัมฤทธิ์ + แรงจูงใจ + ภาวะ flow
- **แต่:** "การ implement ให้ถูกต้องตามหลัก SDT" สำคัญมาก และงานวิจัยยังไม่ชัดว่า implement ที่ดีหน้าตาเป็นยังไง → **อยู่ที่ดีไซน์ของเรา**

## 6. ⚠️ ด้านมืดของรางวัล — ส่วนที่สำคัญที่สุดสำหรับ D-Quest

### 6.1 Overjustification Effect (รางวัลกลบแรงจูงใจภายใน)
- ให้ "รางวัลที่คาดหวังได้-จับต้องได้" กับกิจกรรมที่คนชอบอยู่แล้ว → แรงจูงใจภายในลดลง เพราะคนเปลี่ยนคำอธิบายตัวเองเป็น "ทำเพื่อรางวัล"
- **การทดลองคลาสสิก:** เด็กที่ได้รางวัลสำหรับวาดรูป (ที่ชอบอยู่แล้ว) ภายหลังวาดน้อยลง
- **เฉพาะเจาะจงกับการทำดี:** ทารก 20 เดือนที่ได้ **รางวัลสิ่งของ** สำหรับการช่วยเหลือ ภายหลัง **ช่วยน้อยลง** เทียบกับกลุ่มที่ได้ **คำชม** หรือ **ไม่ได้อะไร**
- **→ ความเสี่ยงตรงหัวใจ D-Quest:** ถ้าเด็กเริ่มมองว่า "เก็บขยะ = ได้ D Coin" แทน "เก็บขยะ = เป็นคนดี/รับผิดชอบ" พอวันไหนไม่มีแต้ม อาจเลิกทำ

### 6.2 ClassDojo — บทเรียนจากแอปจริง (ให้แต้มพฤติกรรม, มี avatar มอนสเตอร์, ใช้ทั่วโลก)
ข้อวิจารณ์หลัก 4 ข้อ + D-Quest รับมือยังไง:
| ปัญหาของ ClassDojo | D-Quest |
|---|---|
| **ประจานต่อหน้า** (โชว์แต้มลบทำเด็กอาย) | เราให้แต้ม **เฉพาะด้านบวก** ไม่มีหักต่อหน้า ✅ · แต่ **กระดานอันดับ = การเปิดเผยสาธารณะ** ต้องระวัง (ดู §6.3) |
| **สอดส่อง** (เด็กกล้าเสี่ยง/สร้างสรรค์น้อยลง เพราะกลัวโดนจับผิด) | เราไม่ติดตามพฤติกรรมลบเชิงสอดส่อง ✅ |
| **พฤติกรรมแสร้งทำเพื่อแต้ม** (ไม่ใช่นิสัยจริง พอเลิกระบบก็เลิกทำ) | **ความเสี่ยงของเรา** — เด็กอาจ "จัดฉากทำดี" เพื่อแต้ม → ครูตรวจช่วยกรอง + ออกแบบให้คุณค่าอยู่ที่ "ความดีจริง" |
| **อคติวัฒนธรรม** (เกณฑ์ความดีอิงค่านิยมกลุ่มหนึ่ง) | เกณฑ์ "ความดี" ของเราควรเปิดกว้าง ไม่ใช่แค่ "เชื่อฟัง/เรียบร้อย" |
- งานวิจัยสรุป: การโชว์แต้มสาธารณะ **"มักไม่เปลี่ยนพฤติกรรมระยะยาว และบางทีแย่ลง"**

### 6.3 Token Economy / PBIS (ระบบแต้มแลกรางวัล — หลักฐานเป็นยังไง)
- หลักฐาน **ผสม** — ลดพฤติกรรมก่อกวนได้ แต่ **ยังไม่ผ่านเกณฑ์ "best practice"** ของ What Works Clearinghouse
- แบบที่ได้ผลกว่า = **"cost-benefit / responsive"** (ปรับตามการตอบสนอง) + รางวัลที่ **เด็กอยากได้จริง** + token ทำหน้าที่ "เตือนให้นึกถึงเป้า"

### 6.4 → หลักออกแบบรางวัลที่ "ไม่ทำลาย" แรงจูงใจ (เอาไปใช้กับ D-Quest)
1. **ทำให้รางวัลเป็น "การยอมรับ/ให้ข้อมูล" ไม่ใช่ "ค่าจ้าง"** — เน้นข้อความแบบ "คุณช่วยให้โรงเรียนน่าอยู่ขึ้น 🌿" มากกว่า "+5 เหรียญ" (ตัวเลขเป็นรอง คำยอมรับเป็นหลัก)
2. **Autonomy:** ให้เด็ก **เลือกเองว่าจะทำดีอะไร** (มีหลายประเภท) ไม่บังคับโควตา
3. **Competence:** เน้น **ความก้าวหน้า/ความเชี่ยวชาญ** (streak, สะสม, ปลดล็อก) มากกว่าตัวเลขแต้มล้วน
4. **Relatedness:** ใช้ **ทีมแผนก/ชุมชน** (เรามีอยู่แล้ว — สองกระดาน) ให้รู้สึกทำเพื่อกลุ่ม
5. **คำชมจากครู > ตัวเลข:** ข้อความอนุมัติของครู (ที่เรามี) มีพลังกว่าที่คิด — งานวิจัยชี้ "คำชมที่บอกว่าเขาเก่ง/มีน้ำใจ" เสริมแรงจูงใจภายใน ส่วน "คำชมเพื่อเอาใจผู้มีอำนาจ" กลับเป็นการควบคุม → **ออกแบบข้อความอนุมัติให้ชมที่ "นิสัย/น้ำใจ" ไม่ใช่แค่ "ได้แต้มแล้ว"**
6. **รางวัลใหญ่แบบ "ไม่คาดคิด/เป็นครั้งคราว"** (unexpected) ทำลายแรงจูงใจน้อยกว่ารางวัลที่ผูกตายตัวทุกครั้ง → ของรางวัลซีซั่นแบบ "เซอร์ไพรส์" ดีกว่าสัญญาตายตัว
7. **ระวังกระดานอันดับ:** เป็นดาบสองคม — สร้าง relatedness/competence ได้ แต่ถ้าเน้น "ประจานผู้แพ้" จะเสีย · ทางแก้: เน้น **"เฉลิมฉลองผู้ทำดี"** + ให้ทุกคนเห็นความก้าวหน้าตัวเอง ไม่ใช่แค่ Top 3

## 7. ความปลอดภัยรูปภาพ (เด็กอัปรูปทำดี)

- มี API ตรวจรูปอัตโนมัติ (Amazon Rekognition, Sightengine, WebPurify) ตรวจ: เปลือย, ความรุนแรง, อาวุธ, ยาเสพติด, ข้อความฝังในรูป
- **แพลตฟอร์มสำหรับผู้เยาว์ควรใช้ "pre-moderation"** (ตรวจก่อนขึ้นจริง) — D-Quest **มีครูตรวจก่อนให้แต้มอยู่แล้ว = pre-moderation โดยมนุษย์ ✅** (ดีกว่าหลายแอป)
- AI ตรวจไม่สมบูรณ์ 100% → ต้องมี **มนุษย์ + ปุ่มรายงาน** เสมอ
- **→ สำหรับเรา:** ปัจจุบันพอแล้ว (ครูเห็นทุกรูปก่อนอนุมัติ + ลบรูปหลังตรวจ) · ถ้าสเกลใหญ่/คิวเยอะค่อยเสริม AI pre-screen (เช่น Sightengine ฟรีโควตาเล็ก) มากรองชั้นแรกก่อนถึงครู

## 8. ข้อเสนอนำไปใช้กับ D-Quest (สรุปลงมือได้)

**ปรับ "ภาษา" ของรางวัล (ทำได้เลย ต้นทุนต่ำ ผลสูง):**
- ข้อความอนุมัติของครู → ชมที่ **น้ำใจ/นิสัย** ("น้ำใจดีมาก ขอบคุณที่ดูแลส่วนรวม") ไม่ใช่แค่ "+5 เหรียญ"
- หน้า home/quest → สื่อสารว่า "ทำเพื่อโรงเรียน/ส่วนรวม" ไม่ใช่ "ทำเพื่อสะสมแต้ม"

**โครงสร้าง (ยึดตามแผนเดิม — งานวิจัยหนุน):**
- คงสองกระดาน (relatedness แผนก) ✅ · คงครูตรวจ (pre-moderation + คำชม) ✅ · คงให้เลือกประเภทความดีเอง (autonomy) ✅
- กาชา Fragment ในอนาคต: **โชว์ pity counter** + ใจดีกว่าเกมเงินจริง + Refinery กันเศษกาก

**ตัวละคร (ตาม north star GL2):**
- เติมตามลำดับ: แยกผมหน้า/หลัง/ข้างซ้าย-ขวา + ตา-ลูกตา-รูม่านตา → UI transform ต่อชิ้น → slider↔palette
- ตัดสินใจเรื่อง **asset วาดมือ vs geometric** (ถ้าอยากได้ลุคอนิเมะแบบ GL2 ต้องมีภาพวาด ไม่ใช่แค่โค้ด)
- ใช้แนวคิด **"avatar = โค้ด export/import ได้"** (เรามี avatar_key อยู่แล้ว)

**ของที่ควรระวัง:**
- อย่าให้กระดานกลายเป็นการ "ประจาน" · อย่าให้ระบบกระตุ้น "แสร้งทำดีเพื่อแต้ม" (ครูตรวจช่วยได้) · เกณฑ์ความดีอย่าแคบแค่ "เรียบร้อย/เชื่อฟัง"
