# D-Quest — สเปกการผลิต Skin (สำหรับปั้มรูปด้วย AI)

> เป้าหมาย: ปั้มรูปยังไงให้ **เสียบเข้าระบบได้ทันที ไม่ต้องแก้โค้ด** · ภูมิปั้มรูป, ผมทำระบบรับ
> หลัก: **ทำสินค้า (skin) ก่อน → ค่อยทำกาชาแจก** · ทุกอย่างในสเปกนี้คือ "แม่พิมพ์" ของสินค้า

---

## 1. โมเดลที่เลือก: "ลุคเต็มตัว + ของแต่งลอยทับ" (แนะนำ)
AI เก่งเรื่อง "วาดตัวละครทั้งตัว" แต่ **ห่วยเรื่องวาดชิ้นส่วนแยกให้ต่อกันพอดีบนโครงเดียว** (เอาผม+เสื้อ+กางเกงที่ปั้มแยกมาต่อกัน = เหลื่อมแน่นอน) เพราะงั้น v1 ใช้แบบนี้:

- **Skin = รูปตัวละครเต็มตัว 1 รูป** (ร่าง + เสื้อผ้า + ผม + หน้า ครบในรูปเดียว) ← นี่คือ "สินค้าหลัก"
- **ของแต่ง/เอฟเฟกต์ = รูปแยกที่ "ลอยทับ"** (หมวก, ปีก, รัศมี, แสงเรือง) วางตามตำแหน่ง ไม่ต้องต่อกับร่างเป๊ะ ← เป็นของกาชาเสริม (เฟสถัดไป)

> แบบ "ผสมชิ้นส่วนอิสระเต็มสูตร GL2" (mix hat ชุด A + เสื้อชุด B) ไว้เฟสหลังมาก เพราะต้องวาดทุกชิ้นบนโครงกระดูกเดียว = งานหิน · ตอนนี้ "ลุคเต็มตัว" คุ้มสุด

---

## 2. ฟอร์แมตไฟล์ (สำคัญสุด — พลาดตรงนี้ใช้ไม่ได้)
- **ไฟล์:** PNG พื้นหลัง**โปร่งใส** (transparent) — **ห้ามมีพื้นหลัง** (ไม่เอาฉาก/สีพื้น)
- **ขนาด canvas:** แนวตั้ง อัตราส่วน ~**3:4 ถึง 5:7** · แนะนำ **1024 × 1450 px** (ใช้เท่ากันทุกรูป)
- **ตัวละคร:** เต็มตัว ยืนตรง หันหน้า · **เท้าอยู่ใกล้ขอบล่าง** (เว้นขอบล่างนิดเดียว) · จัดกึ่งกลางแนวนอน
- **โครง/สัดส่วนเหมือนกันทุก skin:** ปั้มทุกตัวจาก **ท่า/สัดส่วนอ้างอิงเดียวกัน** (ใช้ภาพ reference/pose เดิม) เพื่อให้ทุก skin ยืนบนแท่นได้สวยเท่ากัน + สลับกันแล้วไม่กระโดด
- ระบบจะย่อรูปลงให้พอดีเวที (anchor ที่เท้า) อัตโนมัติ — อัตราส่วนเพี้ยนนิดหน่อยไม่เป็นไร แต่ "เท้าติดล่าง + กึ่งกลาง" ต้องตรง

## 3. สี (ข้อจำกัดของรูป AI)
- รูป AI = **สีตายตัว** เปลี่ยนสีด้วย slider ในเว็บไม่ได้ (raster)
- อยากมีตัวเลือกสี → **ปั้มเป็นหลายรูปแยก** (เช่น `s1_หมีน้ำเงิน`, `s1_หมีแดง`) แต่ละสี = 1 skin

## 4. Set / Rarity (ไว้ผูกกับกาชาทีหลัง)
- **Set** = คอลเลกชันธีมเดียวกัน — เริ่ม **2 set** ก่อน (เช่น Set A "ชุดช่าง", Set B "ชุดพละ")
- แต่ละ skin **ติดระดับ rarity:** N / R / SR / SSR / UR (กาชาจะใช้ค่านี้กำหนดความหายาก/จำนวน fragment)
- 1 set อาจมีหลาย skin + ของแต่งหลายชิ้น — เริ่มน้อยๆ ก่อนได้ (เช่น set ละ 3–5 skin)

## 5. การตั้งชื่อไฟล์ + ที่เก็บ
- **bucket ใน Supabase:** `skins` (public) — เดี๋ยวผมสร้างตอนทำระบบ
- **ตั้งชื่อ:** `skin_<set>_<ชื่อ>_<rarity>.png` เช่น `skin_a_mechanic_sr.png`
- ของแต่ง: `acc_<ตำแหน่ง>_<ชื่อ>.png` เช่น `acc_head_goldcrown.png` (ตำแหน่ง: head/face/back/effect)

## 6. เคล็ดให้รูป AI สม่ำเสมอ (กันเหลื่อม)
- ปั้ม "ร่างเปล่า/โพสอ้างอิง" ก่อน 1 ตัว แล้วใช้เป็น reference ให้ทุก skin ยืนท่าเดียวกัน สเกลเดียวกัน
- ครอปให้เท้าอยู่ระดับเดียวกันทุกรูป (baseline เดียว)
- ของแต่งปั้มบน canvas ขนาดเดียวกัน วางตรงตำแหน่งที่มันควรอยู่ (หมวกอยู่โซนหัว, ปีกอยู่โซนหลัง) แล้ว export โปร่งใส

## 7. ระบบที่ผมจะทำมารับ (ฝั่งโค้ด)
1. ✅ **ตอนนี้:** Character รองรับ "skin รูปเต็มตัว" แล้ว (ใส่ URL รูป → แสดงแทนตัว geometric, ไม่มี skin = แสดงตัว geometric เดิม)
2. **ถัดไปเมื่อมีรูป:** bucket `skins` + ตาราง `skins`(catalog) + `owned_skins`(ของที่มี) + หน้าเลือกสวม skin ในโปรไฟล์
3. **หลังจากนั้น:** กาชา Fragment เต็มสูตร (สุ่มได้ skin/fragment ตาม rarity → Refinery → Pity) ตาม `CHARACTER_GACHA_ARCHITECTURE_th.md`

> สรุปงานของคุณตอนนี้: ปั้ม **2 set, skin เต็มตัว, PNG โปร่งใส, ~1024×1450, เท้าติดล่าง-กึ่งกลาง, ติด rarity** แล้วส่งมา 1 รูปก่อนก็ได้ ผมจะต่อระบบให้เห็นจริงบนเวที แล้วค่อยขยาย
